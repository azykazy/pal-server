# Public IP は起動時作成・停止時削除 (Terraform 管理外)

## 状況

Azure の Standard Public IP は静的のみで、リソースが存在する限り $0.005/h (約$3.65/月) 課金される。
Basic SKU (動的) は 2025-09 に廃止済み。ユーザーは「Discord で接続情報を共有できるなら IP が毎回変わってもよい」と判断。

## 決定

- Public IP リソースは Terraform で管理しない (NIC は IP なしで作成し `ignore_changes = [ip_configuration]`)
- Functions が `/palworld start` で作成・NIC に関連付け、停止時に切り離して削除する
- Spot eviction で IP が残った場合に備え、毎日1回の cleanup タイマーで掃除する

## 理由

停止中の固定費が「ディスクのみ」になり、月30時間運用の総額が約$9.7→約$3.8 (32GBディスク時) に下がる。
動的 IP 運用の AWS (約$8.8) より安くなり、クラウド選定 ([[DEC-20260717-001]]) の結論も強化される。

## トレードオフ

- IP が起動ごとに変わる (接続情報は Discord に都度表示するので運用上の問題なし)
- Terraform の管理外リソースが生まれ、start/stop のコードパスが少し複雑になる
- サブネットの既定送信アクセスは廃止予定のため明示無効化しており、初回プロビジョニングは
  IP が付与されてから systemd timer のリトライで完了する方式にした
