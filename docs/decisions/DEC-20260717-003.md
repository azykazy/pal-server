# Discord 連携はサーバーレス Webhook 方式 + Flex Consumption

## 状況

Discord からサーバーを起動・停止したい。Gateway (WebSocket) 常駐 Bot は常時稼働コストがかかり
「使っている間だけ課金」の要件に反する。

## 決定

- Discord **Interactions Endpoint** (HTTP Webhook) 方式を採用。Ed25519 署名検証を行い、
  3秒制限は deferred 応答 (type 5) + Storage Queue 経由の worker + followup PATCH で回避
- ホスティングは **Azure Functions Flex Consumption (FC1, Node.js 20)**。
  従来の Linux Consumption (Y1) は 2028-09 提供終了予定でレガシー扱いのため
- VM 操作は Function の Managed Identity (Virtual Machine Contributor + Network Contributor)

## 理由

- 常駐プロセス不要で、この用途のリクエスト数なら無料枠内 (実質 $0)
- Microsoft 公式が新規サーバーレスアプリに Flex Consumption を推奨
- Managed Identity によりシークレット (サービスプリンシパルの鍵) を持たずに済む

## トレードオフ

- deferred + queue の2段構成になり関数が増える (interactions / worker / internal-stop / cleanup)
- Flex Consumption は azurerm 4.21+ の専用リソースが必要で、zip デプロイはリモートビルド前提
