# 自動停止は VM 内 cron + Functions への委譲で実装

## 状況

プレイヤー0人が続いたら自動でサーバーを止めたい。プレイヤー数は Palworld の REST API
(`/v1/api/players`) で取れるが、セキュリティ上このポート (8212/tcp) は外部公開したくない。

## 決定

- VM 内の cron (5分毎) が localhost の REST API をポーリングし、0人×30分でセーブ+graceful 停止
- その後の **deallocate と Public IP 削除は Functions の internal-stop エンドポイント (function key 保護) に委譲**する

## 理由

- REST API を外部公開せずに済む (Functions のタイマーで外から人数を見る案は公開が必要)
- VM が自分で IP を切り離すと、その瞬間に外への経路を失い後続の ARM 呼び出し (deallocate) が
  できなくなる。常時接続性のある Functions 側で deallocate + IP 削除を完結させるのが安全
- OS の shutdown だけでは "Stopped" (課金継続) にしかならず、ARM 経由の deallocate が必須

## トレードオフ

- VM に function key を配る必要がある (cloud-init で auto-stop.sh に埋め込み、root 0700)
- internal-stop の呼び出しが失敗した場合に備え、毎日の cleanup タイマーを安全網として持つ
