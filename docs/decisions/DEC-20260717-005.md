# シークレットは Key Vault で管理しローカルに置かない

## 状況

当初はパスワード類を terraform.tfvars (gitignore) に置く設計だったが、
ユーザーから「ローカルに値を持ちたくない」との要望があった。

## 決定

- Azure Key Vault (standard, RBAC) を Terraform で作成し、`server-password` /
  `admin-password` / `discord-webhook-url` を格納する
- 値の投入は apply 後に `scripts/seed-secrets.sh` (az CLI) で行い、**Terraform には値を通さない**
  (tfvars にも tfstate にも残らない)
- Functions は app settings の **Key Vault 参照** (`@Microsoft.KeyVault(SecretUri=...)`) で解決
  (Flex Consumption でのサポートは公式ドキュメントで確認済み)
- VM は Managed Identity + IMDS トークンで起動時に `fetch-secrets.sh` が取得し、
  compose 用の `.env` (root, 0600) を生成する。cloud-init にパスワードが埋まらなくなった

## 理由

- Key Vault の課金は操作 $0.03/1万回 (japaneast, Retail Prices API で確認) のみで、
  この用途 (月数百操作未満) では実質 $0
- 両 Managed Identity に Key Vault Secrets User を付与するだけで鍵レス配布が完結する

## トレードオフ

- apply 後に seed-secrets の一手間が増える (シークレット未投入だとサーバー起動に失敗する)
- Function のホストキー (internal-stop 用) と Storage アクセスキーは引き続き tfstate に残る。
  完全にローカルから排除するには remote state (Azure Storage backend) の導入が別途必要
