# Flex Consumption への zip デプロイの落とし穴

## 概要

Azure Functions Flex Consumption (FC1) への zip デプロイは、従来の方法がことごとく
`InvalidAppSettingsException: The app settings SCM_DO_BUILD_DURING_DEPLOYMENT are not
supported with this SKU` で失敗する。確実なのは OneDeploy API の直接呼び出し。

## 詳細

実際に失敗した経路 (2026-07, azurerm 4.81 / az 2.87):

1. **Terraform `zip_deploy_file`**: デプロイサービス準備待ちの 404 で失敗 (既知 issue)
2. **`az functionapp deployment source config-zip`**: az 自身が Flex 非対応の
   `SCM_DO_BUILD_DURING_DEPLOYMENT` を注入して検証で拒否される
3. **`az functionapp deploy --type zip`** (プレビュー): HTTP 415 で失敗
4. ARM 上で設定を削除・アプリ再起動しても、**SCM (デプロイサービス) が環境変数を
   キャッシュ**していて同エラーが続く。`az functionapp config appsettings set → delete` の
   touch で変更イベントを起こすと解消した

成功した方法 (scripts/deploy-functions.sh に実装):

```bash
TOKEN=$(az account get-access-token --query accessToken -o tsv)
curl -X POST -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/zip" \
  --data-binary @dist/functions.zip \
  "https://<app>.scm.azurewebsites.net/api/publish?RemoteBuild=false"
# その後 /api/deployments/latest を status=4 (成功) までポーリング
```

- リモートビルドは使わず、**node_modules を zip に同梱**する (依存が Pure JS なら安全)
- `RemoteBuild=false` が `SCM_DO_BUILD_DURING_DEPLOYMENT` / `ENABLE_ORYX_BUILD` の代替

## 活用場面

Flex Consumption の Function App にコードをデプロイする全ケース。
CI/CD 化するときも同じ API を使えばよい。
